# 项目进度记录

日期：2026-05-12

## 当前阶段

Listingify 已从 SHEIN 发品闭环推进到平台商品生命周期运营阶段。当前主线不只是“把商品发布出去”，而是围绕已发布/已同步到 SHEIN 的平台商品，形成可同步、可查看、可批量维护、可导出、可追踪操作结果的运营闭环。

当前重点链路：

```text
MDM / 深绘商品档案
  -> SHEIN 发布草稿与发布任务
  -> SHEIN 平台商品列表回捞
  -> SPU 详情、SKC、SKU、销售站点明细落库
  -> 平台商品详情运营动作
  -> 批量供货价维护、状态同步、撤回、操作流水
  -> 运营中心聚合对账和真实数据回归
```

运行数据库已经切到 PostgreSQL，历史 SQLite 数据迁移路径已完成规划和本地兼容修复；后续新增表默认按 PostgreSQL 运行口径设计。

## 已完成能力

### SHEIN 平台商品列表

- 新增 `/api/shein-platform-products` 主模块和 `平台商品列表` 页面。
- 平台商品列表支持服务端分页、搜索、品牌筛选、类目筛选、销售站点筛选和本地持久化。
- 同步商品支持按指定时间范围同步，也支持按款号同步。
- 按时间同步可选择 `按更新时间` 或 `按创建时间`，避免同时暴露多组时间框造成误用。
- 同步列表后会继续补拉 SPU 详情，避免只停留在平台列表接口返回的摘要数据。
- 列表项已展示商品图片、SPU、商品名称、供应商货号、品牌名称、类目名称、供货价、销售站点摘要、同步时间和最近操作状态。
- 品牌名称和类目名称优先从项目内已有编码映射关系解析，不直接把编码暴露给运营。

### SPU 详情和运营动作

- 平台商品详情页支持从列表进入，查看 SPU、SKC、SKU、图片、供货价、售价、状态和原始详情摘要。
- 支持单个 SPU 同步详情、检查可编辑、同步状态、撤回商品、失败操作重试。
- 支持常用字段编辑和高级 JSON 编辑，提交前复用 `spu-info` 基线，尽量保留 SHEIN 平台生成标识。
- 支持拼款/追加变体模板，提交前接入商家 SKU 查重。
- 支持平台商品列表页和 SPU 详情页分别批量更新供货价。
- 支持表格导入批量更新供货价，提供模板下载、上传解析、预览和按 SPU/SKC 聚合提交。

### 销售站点视图

- 已从 `/open-api/goods/spu-info` 返回的 `shelfStatusInfoList` 展开销售站点明细。
- 同时兼容 SPU 原始 payload 和各 SKC 原始 payload 中的站点信息，尽量保留真实平台返回。
- 平台商品列表显示 `上架 X 站` 或 `未上架`，点击后弹窗查看具体站点。
- 销售站点明细包含站点、上架状态、商品链接、首次上架时间、最近上架时间。
- 平台商品列表支持按销售站点筛选。
- 当前站点能力是只读运营视图，只表示 SHEIN 商品在哪些站点销售，不代表系统具备修改站点上架关系的能力。

### 导出和表格能力

- 平台商品列表支持按当前筛选条件导出全部本地平台商品。
- 导出文件包含主 Sheet `平台商品列表`，销售站点字段保留统计摘要，例如 `上架 3 站`、`未上架`。
- 导出文件新增 Sheet `销售站点明细`，按 `SPU + 销售站点` 一行一条展开。
- `销售站点明细` 字段包括：`SPU`、`商品名称`、`供应商货号`、`品牌名称`、`类目名称`、`供货价`、`销售站点`、`上架状态`、`首次上架时间`、`最近上架时间`。
- 前端导出下载逻辑改为显式 Blob 链接，兼容浏览器下载行为。

### P1 运营中心

- 新增并接入平台标识对账、商家 SKU 查重、条码尺码、条码打印、成本价涨价原因、审核状态聚合和真实数据回归日志。
- `发布任务` 已升级为 `发布与审核任务` 主操作台，可查看发布和审核状态。
- `SHEIN运营中心 > 审核状态` 作为跨发布任务、平台商品和生命周期操作的聚合视图。
- 涨价原因按真实接口优先、本地枚举兜底的方式缓存，等待真实账号进一步确认接口开放性。
