## 2. 文档目的

本文档用于统一 Listingify 后续开发方向，把旧项目的中台化思维与当前已经跑通的 SHEIN 发品能力合并成一版可执行的产品口径。

本文档回答四个问题：

1. 当前产品已经具备什么能力。
2. 后续开发必须遵守什么业务边界。
3. 下一阶段优先建设哪些产品能力。
4. 哪些能力只预留，不在当前阶段实现。

## 3. 背景与问题

森马跨境上新已经具备三类基础输入：

1. `MDM`：提供 SPU、SKC、SKU、品牌、类目、颜色、尺码、吊牌价、条码等主数据。
2. `深绘`：提供标题、详情页、图片素材、尺码表、字段池、内容包等运营内容。
3. `Excel / 人工规则`：提供毛重、尺码转换、包装规则、低倍率清单、价格折扣、类目映射等业务规则。

Listingify 当前已经把这些输入串进 SHEIN 全托管发品链路，并完成真实单款发布。但从单款跑通到批量可用，还存在以下问题：

1. 批次级发布、失败归因、批量重试和状态同步还需要稳定化。
2. 发布失败、审核驳回、图片转换失败、字段阻断等异常仍散落在任务和校验结果中，需要形成统一异常闭环。
3. 字段来源、人工覆盖、规则命中、发布版本之间的血缘关系需要更明确，避免后续多平台扩展时失控。
4. SHEIN 相关字段必须继续与商品事实层隔离，不能反向污染 MDM/深绘聚合模型。
5. TEMU 等多平台要预留边界，但不能在 SHEIN 批量闭环稳定前提前实现。

## 4. 产品定位

Listingify 不是完整 ERP，也不是单纯 SHEIN 接口工具。当前定位是：

1. 对业务：SHEIN 全托管上新的生产工作台，替代 Excel 拼表和平台后台逐条发品。
2. 对系统：商品事实层之上的平台刊登中台，沉淀发布包、规则、任务、异常和审计能力。
3. 对未来：多平台刊登适配的基础骨架，但当前只交付 SHEIN。

核心边界：

1. 商品事实层负责沉淀平台无关的商品真相。
2. SHEIN 执行层负责把商品真相转换为 SHEIN 可提交、可审核、可追踪的发布动作。
3. 发布包和任务中心负责把一次操作变成可审计、可重试、可追责的生产记录。

## 5. 当前已跑通能力基线

以下能力作为 2026-05-01 后续开发的事实基线。

### 5.1 数据底座

1. MDM 商品主数据已按 `SPU -> SKC -> SKU` 落库。
2. 深绘内容包已落库，覆盖内容包、款色、SKU、图片素材、详情页、尺码表和字段池。
3. 商品档案聚合 MDM 与深绘数据，保留源头 payload，不混入 SHEIN 业务字段。
4. 图片素材库可按 SPU/SKC、图片类型、来源平台、尺寸和文件大小查看。

### 5.2 SHEIN 元数据与规则

1. SHEIN 类目树、发布字段、属性模板、属性枚举、图片规则和尺码要求已可同步、导入和查询。
2. 类目映射支持 `MDM 中类 + 小类 + 性别 + 年龄段` 组合规则和 AI 推荐审核。
3. 尺码转换、包装规则、价格规则、低倍率清单和 SKU 毛重表支持维护。
4. SKU 毛重已修正为 SKU 维度存取。

### 5.3 SHEIN 商品分桶

1. 商品档案可挑选进入 SHEIN 商品分桶。
2. 分桶商品承载 SHEIN 平台字段，与商品档案隔离。
3. 支持类目、英文标题、价格、包装、图片状态、字段完整度和发布历史管理。
4. 支持 AI 结合商品档案、深绘字段、SHEIN 类目和枚举上下文推荐类目和字段值。
5. 支持从分桶商品派生多个独立发布草稿。

### 5.4 SHEIN 发布草稿箱

1. 草稿列表支持图片、SPU、款色、品牌、类目、版本、状态、字段完整度、阻断项和更新时间展示。
2. 支持草稿创建、复制新版本、暂停/恢复、归档/删除、批量校验和批量发布。
3. 单款草稿详情按 `SPU -> SKC -> SKU` 维度维护字段。
4. SHEIN 类目选择支持树形多级选择弹窗。
5. 尺码表以 SHEIN 类目模板为准，发布尺码以转换后的 SHEIN 尺码为准。
6. 图片模块支持本地上传、素材库选择、编辑、删除、确认和平台图片规则合规判断。

### 5.5 发布任务与批次

1. 上新批次可组织多个 SHEIN 发布草稿。
2. 发布任务列表支持分页、状态筛选、批量搜索和重试入口。
3. 发布任务详情支持查看请求 payload、响应、traceId、失败原因、关联草稿和版本。
4. SHEIN 发布接口链路已接入，具备 payload 构建、提交、失败回写、任务记录和审核状态同步能力。
5. `201122104105 / 中黄30435` 已完成真实推送并成功进入 SHEIN 后台。

### 5.6 系统管理

1. 已加入登录鉴权、会话 Cookie、RBAC 权限和默认管理员初始化。
2. 已加入用户管理。
3. 已加入平台对接配置，支持 SHEIN 店铺/应用密钥、环境、业务模式、默认账号和测试状态，接口响应脱敏。
4. 已加入操作日志，记录登录、发布、用户管理、平台配置等关键动作。
5. 同步任务页面整合 MDM、深绘、SHEIN 元数据和发布任务记录。

## 6. 产品目标

### 6.1 当前阶段目标

当前阶段目标是把 SHEIN 从“单款真实跑通”推进到“批量可运营”。

必须达成：

1. 批次级批量发布可稳定执行。
2. 批量发布支持部分成功、部分失败。
3. 失败项可按原因聚合、定位、修复和重试。
4. 审核状态可自动或半自动同步到批次、草稿和任务。
5. 发布包、请求响应、版本差异和平台身份可追溯。
6. 阻断项能贴近字段展示，并可跳转修复。

### 6.2 平台化目标

在不提前开发 TEMU 发布的前提下，固化以下平台化原则：

1. 核心商品模型不绑定 SHEIN 字段。
2. 平台特有字段进入 `channel_*`、`shein_*` 或 `listing_*` 模型。
3. 发布任务、发布版本、平台身份、操作日志按平台维度统一建模。
4. 平台适配器只负责协议、payload 和状态转换，不反向修改商品事实定义。
5. TEMU 只能在 SHEIN 批量闭环稳定后进入 adapter 验证阶段。

## 7. 非目标

当前阶段不做以下内容：

1. TEMU 真实发布。
2. 跨平台一键发布。
3. SHEIN 订单、履约、库存、采购、结算、财务对账。
4. 完整合规资产中心和合规流程。
5. 自动生成商品图片或全自动修图。
6. 自动生成全部平台属性映射。
7. 跨平台统一上下架。
8. 跨平台统一直接调价。
9. 替代 MDM 或深绘的源头维护职责。

## 9. 核心概念

| 概念 | 定义 |
| --- | --- |
| 商品事实层 | MDM、深绘和人工规则融合后的平台无关商品数据 |
| SHEIN 商品分桶 | 商品进入 SHEIN 发布准备池后的平台字段清洗记录 |
| 发布草稿 | 面向 SHEIN 某账号、某发布单元生成的待发布资料 |
| 发布包 | 某次发布前冻结的数据、价格、图片、属性、尺码和 payload 快照 |
| 发布版本 | 草稿每次保存、发布、重提形成的版本记录 |
| 发布任务 | 一次提交、状态同步或重试的异步执行记录 |
| 任务分项 | 批量任务中某个草稿、SKC 或 SKU 的执行结果 |
| 异常 | 阻断、失败、审核驳回、同步失败等需要人工处理的事项 |
| 平台身份 | SHEIN 返回的 `spu_name`、`skc_name`、`sku_code` 等平台对象 ID |
| 字段血缘 | 字段值来自 MDM、深绘、规则、AI 或人工覆盖的来源记录 |

## 10. 核心业务流程

### 10.1 SHEIN 批量上新主流程

```text
MDM/深绘同步
  -> 商品档案确认
  -> 进入 SHEIN 商品分桶
  -> AI/规则推荐类目和字段
  -> 生成 SHEIN 发布草稿
  -> 补齐 SPU/SKC/SKU 字段
  -> 图片确认和转换
  -> 价格包装确认
  -> 发布前校验
  -> 生成发布包版本
  -> 批次级提交发布任务
  -> SHEIN 审核状态同步
  -> 成功归档或异常修复重提
```

### 10.2 异常处理流程

```text
发布前校验 / 图片转换 / SHEIN 接口 / 审核同步产生错误
  -> 系统归一化错误类型
  -> 生成或更新异常记录
  -> 异常工作台按责任角色和对象层级聚合
  -> 运营跳转到草稿字段、图片、价格或任务详情修复
  -> 修复后生成新版本
  -> 基于新版本重试
  -> 异常关闭并审计留痕
```

### 10.3 批量发布状态流程

```text
草稿可发布
  -> 加入上新批次
  -> 批量生成发布任务
  -> 每个草稿独立提交
  -> 批次状态汇总为成功 / 部分成功 / 失败 / 审核中
  -> 失败项按原因聚合
  -> 可批量重试失败项
```

## 11. 功能需求

### 11.1 商品档案

目标：保持平台无关的商品事实视图。

需求：

1. 展示 MDM 和深绘聚合后的 SPU/SKC/SKU 数据。
2. 展示源头 payload、同步批次、更新时间。
3. 展示图片、详情页、尺码表和字段池。
4. 不允许在商品档案中直接维护 SHEIN 发布字段。
5. 支持进入 SHEIN 商品分桶。

验收标准：

1. 运营可以查看源头数据是否完整。
2. SHEIN 字段不会写回商品事实表。

### 11.2 SHEIN 商品分桶

目标：承载 SHEIN 发布准备字段，并与商品事实层隔离。

需求：

1. 支持从商品档案挑选商品进入分桶。
2. 支持批量款号搜索、筛选、分页。
3. 支持 SHEIN 类目、标题、价格、包装、图片状态、字段完整度维护。
4. 支持 AI 推荐类目和字段值。
5. 支持从同一商品派生多个发布草稿。
6. 支持展示最新草稿、最新发布状态和发布历史。

验收标准：

1. 分桶字段变化不影响 MDM/深绘事实数据。
2. 同一商品可创建多个独立草稿版本。

### 11.3 SHEIN 发布草稿箱

目标：作为 SHEIN 发布前的主生产界面。

需求：

1. 草稿列表展示状态、阻断项、字段完整度、图片、版本和更新时间。
2. 支持批量校验、批量发布、暂停、恢复、归档、删除。
3. 单款详情按 SPU/SKC/SKU 维护字段。
4. 支持 SHEIN 类目树选择，选择后刷新字段、图片规则和尺码模板。
5. 支持字段来源展示：MDM、深绘、规则、AI、人工。
6. 支持人工覆盖字段并记录修改人、时间、原值和新值。
7. 支持保存草稿生成新版本。
8. 支持发布前预览发布包摘要。

验收标准：

1. 草稿详情可完成单款发布所需字段维护。
2. 每次保存或发布都有版本记录。
3. 运营能看到字段来自哪里、被谁改过。

### 11.4 图片素材与转换

目标：把 MDM/深绘/本地上传/素材库图片转成 SHEIN 可用素材。

需求：

1. 草稿详情支持本地上传图片。
2. 草稿详情支持从素材库选择图片。
3. 素材选择默认优先展示 TMALL/VIP 商品图，默认排除商详模块和商详截图。
4. 按 SHEIN 图片规则检查尺寸、大小、图片类型和必填情况。
5. 支持图片确认、删除、编辑、排序。
6. 图片上传/转换结果写入草稿图片记录。
7. 图片转换失败生成阻断项和异常。
8. 支持复用已转换的平台图片 URL。

验收标准：

1. 发布前所有必填图片都有 SHEIN 可用 URL。
2. 不合规图片不能静默进入发布 payload。

### 11.5 价格、包装和毛重

目标：确保 SKU 级商业字段在发布前被确认。

需求：

1. 供货价、毛重、包装长宽高按 SKU 维度维护。
2. 支持价格规则、低倍率清单和手工覆盖。
3. 支持包装规则按商品上下文匹配。
4. 价格字段必须人工确认后才能发布。
5. 发布时固化价格、折扣、币种、毛重和包装快照。
6. 后续规则变化不影响已发布版本。

验收标准：

1. 未确认价格的 SKU 被发布前校验阻断。
2. 发布任务可回查当时的价格和包装快照。

### 11.6 发布前校验

目标：在调用 SHEIN 接口前识别阻断项和警告项。

需求：

1. 校验 SHEIN 叶子类目、`product_type_id` 和发布字段规范。
2. 校验必填商品属性、销售属性、尺寸属性和枚举范围。
3. 校验 SKC 图片、方形图、色块图等图片规则。
4. 校验 SKU 商家 SKU、条码、尺码、供货价、毛重和包装。
5. 校验图片是否已转换为 SHEIN 可用 URL。
6. 校验同一店铺供应商 SKU 是否重复。
7. 阻断项必须包含对象层级、字段、原因、建议动作和跳转修复入口。
8. 警告项不阻断发布，但必须展示。

验收标准：

1. 阻断错误 100% 有字段级原因。
2. 发布接口不承担基础字段缺失校验。

### 11.7 批次与批量发布

目标：把单款发布能力扩展为批量运营能力。

需求：

1. 上新批次可包含多个发布草稿。
2. 批次列表展示款数、SKC 数、SKU 数、待补齐、校验失败、待发布、发布中、审核中、通过、失败。
3. 批次详情展示草稿明细、任务汇总和失败原因聚合。
4. 支持批次级批量校验。
5. 支持批次级生成发布任务。
6. 支持部分成功、部分失败。
7. 支持失败项按原版本或新版本重试。
8. 支持批次级状态同步。

验收标准：

1. 一个草稿失败不影响同批次其他草稿提交。
2. 批次可看到成功、失败、审核中和待处理数量。

### 11.8 发布任务中心

目标：统一追踪发布、重试和审核同步。

需求：

1. 任务列表支持按状态、批次、平台、款号、失败类型筛选。
2. 任务详情展示请求 payload、响应 payload、traceId、失败原因、平台版本和关联草稿。
3. 任务记录失败分类、失败指纹、是否可重试、重试次数和下次重试时间。
4. 支持失败任务重试。
5. 支持审核状态手动同步和定时同步。
6. 支持平台身份回写。

验收标准：

1. 发布任务可追踪率 100%。
2. 发布成功任务平台身份回写率 100%。

### 11.9 异常工作台

目标：把失败任务和阻断项从日志中提出来，形成业务闭环。

需求：

1. 统一聚合发布前阻断、发布失败、图片转换失败、审核驳回、状态同步失败、配置错误、限流错误。
2. 支持按平台、批次、品牌、店铺、责任角色、错误类型、状态筛选。
3. 异常详情展示原始错误、归因说明、修复建议、关联任务、关联草稿和版本。
4. 支持关闭异常、重新打开、修复后重试。
5. 所有处理动作写入操作日志。

验收标准：

1. 失败项不只存在于接口响应或日志。
2. 运营能从异常跳转到可修复位置。

### 11.10 审计与安全

目标：确保关键动作可追责，密钥和权限可控。

需求：

1. 平台密钥脱敏展示。
2. 发布、重试、配置修改、用户管理、AI 覆盖、人工覆盖字段写入操作日志。
3. 发布、批量重试、密钥修改等高风险动作二次确认。
4. RBAC 权限至少覆盖用户管理、平台配置、规则维护、发布操作、日志查看。
5. 审计日志不得由普通用户删除。

验收标准：

1. 关键链路能追到人、时间、对象、动作和结果。
2. 非授权用户不能执行发布和密钥配置。

## 15. 风险与处理

| 风险 | 处理策略 |
| --- | --- |
| SHEIN 类目和属性规则变化 | 定期同步，发布前校验关键类目规则版本 |
| 批量发布触发限流 | 任务队列分批执行，记录可重试错误和退避时间 |
| 图片转换失败 | 发布前阻断，允许重新上传、重新转换和复用已转换 URL |
| 审核驳回原因不结构化 | 保存原始返回，人工归因，沉淀常见错误分类 |
| MDM 与深绘字段冲突 | 保留字段血缘和优先级规则，人工覆盖留痕 |
| 价格规则误用 | 发布前人工确认，发布时固化快照 |
| TEMU 差异过早污染模型 | 当前只做 adapter 边界和字段验证，不做真实 TEMU 发布 |
