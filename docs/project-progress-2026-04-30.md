# 项目进度记录

日期：2026-04-30

## 当前阶段

Listingify 已从 SHEIN 元数据和数据底座阶段推进到发品闭环可操作阶段。当前主链路已经串起：

```text
MDM / 深绘同步
  -> 商品档案留存源头数据
  -> SHEIN 商品分桶做平台字段清洗
  -> SHEIN 发布草稿箱做草稿版本和校验
  -> 上新批次组织批量发布
  -> 发布任务追踪提交、失败、重试和状态同步
```

当前实现以 SHEIN 全托管为首个平台，核心商品档案、平台元数据、发布草稿、发布任务、平台对接配置和操作日志均保留多平台扩展点。

## 已完成能力

### 数据底座

- MDM 商品主数据已按 `SPU -> SKC -> SKU` 落库。
- 深绘内容包已落库，覆盖内容包、款色、SKU、图片素材、详情页、尺码表和字段池。
- 商品档案聚合 MDM 与深绘数据，保留原始源头数据，不混入 SHEIN 业务字段。
- 图片素材库可按 SPU/SKC、图片类型、来源平台、尺寸和文件大小查看。

### SHEIN 元数据与规则

- SHEIN 类目树、发布字段、属性模板、属性枚举、图片规则和尺码相关要求已可同步、导入和查询。
- 类目映射支持 `MDM 中类 + 小类 + 性别 + 年龄段` 组合规则和 AI 推荐审核。
- 尺码转换、包装规则、供货价/价格规则、低倍率清单和 SKU 毛重表支持表格导入、导出、批量搜索和增删改查。
- SKU 毛重已修正为 SKU 维度存取，不再按 SPU 均值覆盖。

### SHEIN 商品分桶

- 商品档案可挑选进入 SHEIN 商品分桶。
- 分桶列表支持服务端分页、批量款号搜索、品牌/类目/状态/图片状态/完整度筛选。
- SHEIN 平台字段与商品档案隔离，支持类目、英文标题、价格、包装、图片状态、字段完整度和发布历史管理。
- 支持 AI 结合商品档案、深绘字段、SHEIN 类目和枚举上下文推荐类目和字段值。
- 支持从分桶商品派生多个独立发布草稿，每个草稿有独立版本和状态机。

### SHEIN 发布草稿箱

- 菜单和页面已统一为「SHEIN 发布草稿箱」。
- 草稿列表支持图片、SPU、款色、品牌、类目、版本、状态、字段完整度、阻断项和更新时间展示。
- 支持草稿创建、复制新版本、暂停/恢复、归档/删除、批量校验和批量发布。
- 批量发布遇到阻断时，会弹出快速调整界面，支持修复阻断字段、SKU 毛重和图片确认。
- 单款草稿详情按 `SPU -> SKC -> SKU` 维度组织：
  - SPU：平台类目、标题、商品属性、发布字段和整体保存。
  - SKC：发布款色、款色图片、颜色属性、图片规则、是否发布。
  - SKU：MDM SKU、发布尺码、供货价、SKU 毛重、包装长宽高和是否发布。
- SHEIN 类目选择改为树形多级选择弹窗。
- 尺码表以 SHEIN 类目模板为准，发布尺码以转换后的 SHEIN 尺码为准，并与勾选 SKU 对齐。
- 图片模块支持本地上传、素材库选择、编辑、删除、确认和平台图片规则合规判断。

### 图片选择与导入

- 草稿详情页支持从图片素材库选择图片。
- 素材弹窗可按图片类型筛选，默认排除「商详模块」「商详截图」。
- 素材弹窗可按来源平台筛选，默认选择 TMALL 和 VIP。
- 图片卡片显示来源平台、图片类型、尺寸和文件大小。
- 合规图片优先展示，支持一键剔除不符合 SHEIN 图片尺寸或大小要求的素材。
- 本地上传图片会记录到 `listing_asset`，并可在草稿图片模块确认、编辑和删除。

### 发布任务与批次

- 上新批次用于组织多个 SHEIN 发布草稿，可查看批次下草稿数量、SKC/SKU 范围、阻断项、发布中、审核中、通过和失败数量。
- 发布任务列表支持服务端分页、状态筛选、批量搜索和重试入口。
- 发布任务详情支持查看请求 payload、响应、traceId、失败原因、关联草稿和版本。
- SHEIN 发布接口链路已接入，具备 payload 构建、提交、失败回写、任务记录和审核状态同步能力。
- `201122104105 / 中黄30435` 已完成真实推送并成功进入 SHEIN 后台，后续不再作为发布联调待办项。

### 系统管理

- 已加入登录鉴权、会话 Cookie、RBAC 权限、默认管理员初始化。
- 已加入用户管理，支持用户创建、编辑、启停、角色分配和重置密码。
- 已加入平台对接配置，支持 SHEIN 店铺/应用密钥、环境、业务模式、默认账号和测试状态，接口响应脱敏。
- 已加入操作日志，记录登录、发布、用户管理、平台配置等关键动作。
- 同步任务页面整合 MDM、深绘、SHEIN 元数据和发布任务记录。

## 与 spec 的剩余差距

- 单款真实发布已跑通，下一步差距集中在批量发布稳定性、失败归因聚合和自动化状态同步。
- 审核状态已支持通过 `/open-api/goods/query-document-state` 手动同步并回写页面，后续需要补定时轮询、通知和批次级同步。
- 图片上传/转换已进入发布链路，后续需要完善平台图片标识复用、转换结果对比、失败重试和素材质量报表。
- 价格流程仍以内部字段维护为主，后续需要接入 SHEIN 价格确认/价格任务的完整状态机。
- 多平台适配器接口已经在模型层预留，但 TEMU adapter 和跨平台字段映射尚未实现。
- 发布前阻断项目前覆盖核心必填字段，后续需要补全全部 SHEIN 类目字段的类型化校验和枚举一致性校验。

## 下一步计划

1. SHEIN 批量发布稳定化：把已成功的单款发布能力扩展到批次级提交、批次级同步、失败归因聚合和批量重试。
2. 发布状态自动化：在现有手动同步基础上增加定时轮询、状态变更通知、审核驳回阻断项生成和修复后重提。
3. 图片转换治理：在已有上传/转换基础上补充平台图片标识复用、转换结果对比、失败重试、素材质量报表和不合规素材清理策略。
4. 阻断修复增强：支持更多字段类型、跨 SKC/SKU 批量应用、字段变更差异预览和修复历史记录。
5. 规则中心治理：为尺码、包装、供货价、SKU 毛重、低倍率清单补充导入批次审计、冲突检测和回滚。
6. 多平台抽象与 TEMU 准备：沉淀 `PlatformAdapter` 接口，拆出 SHEIN 适配器，实现 TEMU 元数据验证、草稿字段映射和发布任务骨架。
